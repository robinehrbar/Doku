Internet

Als allererstes, bevor man mit dem Scripten anfängt sollte man überhaupt wissen wie das Internet funktioniert, um später alle Zusammenhänge verstehen zu können. Das Internet ist eigentlich nichts anderes als ein sehr grosses Netzwerk, wie jedes andere Netzwerk verbindet das sogenannnte World Wide Web (www) Computer und andere Geräte miteinander. Ohne das Internet wäre die heutige Welt - so wie sie ist - unvorstellbar. Das Internet arbeitet mit IP Adressen, um alle Geräte identifizieren und um mit denen kommunizieren zu können.

HTML

HTML kurz für HyperText Markup Language, bezeichnet die heutige standard Markup Sprache für Websiten und deren Programme. HTML beschreibt die Struktur einer Website und verwaltet damit ihr Erscheinungsbild und Aussehen. Bei HTML redet man grundsätzlich von Elemten, und eben nicht wie Viele denken von Befehlen. Diese Elemente bestehen immer aus Tags. Entweder bestehen die Elemente aus einem einzigen Tag(leeres Element sprich ohne Inhalt) oder aus einem öffnendem und einem schliessendem Tag(mit Inhalt). In den Tags selbst hat es teilweise Attribute, diese bestimmen gewisse Eigenschaften über ein Element.

Struktur

Im Prinzip funktioniert die Strukturierung ähnlich wie bei dem uns allen bekannten Microsoft Word. Nämlich hat man verschiedene Formate für jeweils eine Überschrift oder bestimmten Absatz. Zusätzlich unterteilt man bei HTML noch in header, body und footer, um später einfacher formatieren zu können.

Text

Bei HTML gibt es 6 verschiedene Überschriften, nämlich von h1 bis h6, wobei h1 die Grösste und h6 folglich die Kleinste ist. Die normalen Absätze werden zwischen p tags geschrieben. Ausserdem werden sie mit b fett, mit i kursiv, mit del durchgestrichen, mit ins unterstrichen, mit sup hochgestellt und und mit sub tiefgestellt. Ausserdem kann man mit br einen Absatz einfügen, das Gleiche gilt für hr, nur der Unterschied ist, dass bei hr noch eine Linie dazwischen ist. Auch interessant ist abbr, damit kann man nämlich beim hovern über dem Text einen kleinen eingeblendeten Text sehen.

Listen

Grundsätzlich werden Listeneinträge mit li gemacht, diese Einträge sind dann entweder in geordneten Listen(ol), ungeordneten Listen(ul) oder in Defintionslisten(dl)

Links

Links sind typische Elemente die ein Attribut brauchen, in dem a Element wird ein href Attribut mit einer Website versehen, als Elementinhalt schreibt man dann einfach das was man anklicken soll, et voilà der Link ist fertig. Diese Links können entweder auf einen verwandten Ordner(relativer Link) oder auf eine komplette Adresse, zum Beispiel https://www.google.com verweisen(absoluter Link). Fügt man dem a Element noch ein target="_blank" Attribut hinzu, wird der Link in einem neuen Tab geöffnet.

Bilder

Ähnlich wie bei den Links, referieren auch die Bilder auf absolute oder relative Links, jedoch ist es hier ein img Element anstelle des a Elements bei den Links. Mit alt="Beispieltext" kann zusätzlich noch ein Infotext zu dem Bild hinzugefügt werden, welcher auch zur Suchmaschinenoptimierung hilft. Mit height und width kann Höhe und Breite des Bildes in Pixel bestimmt werden. Am besten eignen sich png, jpg oder auch gif als Bildformate.

Video & Audio

Hier ist es auch nicht so unterschiedlich zu Bilder / Links, ausser dem Üblichen kann man hier noch zusätzlich autoplay, loop, controls und preload einstellen. Autoplay lässt das Video automatisch abspielen, Loop zwingt eine automatische Wiederholung auf, Controls fügt browserspezifische Steuerungselemente hinzu und Preload lässt das Video schon im vorhinein laden.

CSS

CSS wendet Formatierungsregeln auf HTML-Elemente an, sie legen dadurch fest, wie der Inhalt der Elemente angezeigt werden soll. Eine CSS-Regel besteht aus zwei Teilen: einem Selektor und dessen Deklaration. Selektoren geben an, auf welches Element/Klasse/Id die Regel angewandt werden soll. Deklarationen hingegen bestimmen wie die Elemente formatiert werden sollen.

CSS ins HTML einbinden

Man sollte immer ein externes CSS Sheet verwenden, um das Ganze besser unterteilen zu können und um Verwechslungen zu vermeiden. Um das zu machen muss man zuerst ein CSS File erstellen, am Besten speichert man dieses auch im selben Ordner wie die HTML Datei ab. Jetzt kann man schon das CSS File einbinden, dass macht man mit einem link Element, dieses leere Element hat drei Attribute, nämlich href, type und rel. Bei href kommt einfach der absolute oder relative Pfad zur CSS Datei rein, bei type schreibt man type="text/css" und zum Schluss noch rel="stylesheet", wobei die Reihenfolge eigentlich generell keine Rolle spielt.

Selektoren

Man kann auch mehr als nur einem Selektor Regeln zuweisen, dass macht es oft leichter, wenn man ganzen Gruppen oder Klassen bestimmte Eigenschaften vergibt. Dazu gibt es mehrere Varianten, welche unterschiedliche Methoden nutzen:

Universalselektor
Gilt für alle Elemente im Dokument (*)
Typselektor
Gilt für alle übereinstimmende Elementnamen (h1, h2, h3)
Klassenselektor
Gilt für Elemente, deren Attribut class einen Wert hat, der mit dem hinter dem Punkt angegebenen Wert übereinstimmt, wenn noch ein Element vor dem Punkt ist, gilt es nur für diese Elemente mit dieser Klasse (.note / p.note)
ID-Selektor
Gilt für Elemente, deren Attribut class einen Wert hat, der mit dem hinter dem Punkt angegebenen Wert übereinstimmt (#introduction)
Kindselektor
Gilt für Elemente, die dem angegebenen Element unmittelbar untergeordnet sind (li>a)
Nachkommenselektor
Gilt für Elemente, die dem angegebenen Element untegeordnet sind (p a)
Selektor für benachbarte Geschwister
Gilt für Elemente, die dem angegebenen Element direkt gleichgeordnet sind (h1+p)
Allgemeiner Geschwisterselektor
Gilt für Elemente, die dem angegebenen Element gleichgeordnet sind, auch wenn sie nicht direkt aufeinander folgen (h1~p)

Farbe

Da wir jetzt wissen wie man Regeln auf einen oder mehrere Selktoren anwendet, können wir mit einer der verbreitesten Regel anfangen, nämlich mit der Eigenschaft color. Nach color: kann man entweder einen der 147 vordefinierten Farbnamen, RGB Werte (0-255) oder auch Hex-Codes (00-ff) verwenden. Alle bewirken im Grunde das Gleiche, nur dass RGB und Hex ein wenig komplizierter aber dafür auch etwas vielfältiger ist. Diese drei Arten von Werten kann man auch nach background-color: verwenden, entsprechend ist das dann die Hintergrundfarbe des Textes.

Text

Für Anpassungen der Schriftgrösse
font-weight
Um den Text fett zu machen
font-style
Um den Text kursiv zu machen
text-decoration
Um den Text mit Linien oder Effekten zu versehen
text-align
Um den Text links-, rechts- oder mittelbündig zu machen
:hover
Um den Text generell anders erscheinen zu lassen wenn man mit der Maus über den Text fährt

Bilder

Ebenso bei den Bildern gibt es eine Reihe an Deklarationen, ich werde auch wieder auf die Wichtigsten eingehen:

float
Um das Bild nach links(left) oder rechts(right) fliessen zu lassen
width
Um die Bildbreite anzupassen
height
Um die Bildhöhe anzupassen
margin
Um den Abstand zu anderen Elementen zu definieren
Ausserdem kann mit background-image ein Bild als Hintergrund verwendet, dieser kann dann noch mit weiteren Deklarationen versehen werden, um ein besseres Erscheinungsbild zu ermöglichen. Dafür gibt es repeat, fixed, scroll, position und viele Weitere.

Media Queries

Mit @media könnnen bestimmete Deklarationen einem oder mehreren Elementen zugewiesen werden, sobald eine gewisse Bedingung erfüllt ist. Die hierbei verbreiteste Variante ist mit max-width: 768px, nämlich kann man damit Deklarationen bestimmen, die nur gelten sollen sobald das Fenster eine gewisse Breite überschritten hat, in dem Fall wäre dass, wenn der Bildschirm maximal 768 Pixel breit ist.

Praxis

Sobald einem die Theorie von HTML und CSS etwas geläufig wurde, kann man sich ans schreiben seiner ersten Website machen, allerdings sollte man auch dort noch einiges beachten. Bevor man anfängt sollte man unbdeingt plannen wie man sich die Website vorstellt. Einerseits mithilfe eines Wireframes kann man eine Skizze der Website machen auf spielerische Art und Weise, dafür eignet sich am besten die Website namens gomockingbird. Ausserdem sollte man die Struktur der Website in einem Sitemap festhalten, dort würde ich euch die gloomaps Website empfehlen.

Layout in HTML

Um während des Scripten nicht den Überblick zu verlieren und um ein sinnvolles Element zu haben welches man später im CSS definieren kann, würde ich Allen empfehlen, seinen body (Hauptteil) in header, main und footer zu unterteilen.
Auch eine schöne Darstellung kann später helfen sich im Code zurechtzufinden, vorallem wenn andere Leute sich den Code anschauen. Wenn man es noch leichter für aussenstehende Personen machen will, kann man mit (kleiner als)!-- Beispieltext --(grösser als) einen Kommentar dem Code hinzufügen, welcher dann letztendlich auf der Website nicht ersichtlich sein wird.
Für eine optimierte Gliederung ist div wie geschaffen. Damit kann man einerseits zusammengehörige Elemente besser gruppieren, andererseits kann man ihnen ganz einfach und ohne Probleme Klassen zuweisen um später im CSS die Formatierung der Website um Einiges zu erleichtern.

Validierung

Wenn man fertig mit dem Schreiben ist und die Website auch so wie man will funktioniert und aussieht, kann jedoch das validieren nie schaden. Den auf der Validierungsseite von W3C können häufig noch invalide Teile des Codes identifiziert werden. Das beheben dieser, führt zu reduzierter Anzahl an Fehlern in sämtlichen Browsern und schliesst jede Website sauber ab.

Testprotokoll

Leider sind wir immer noch nicht ganz durch, ganz am Schluss sollte man seine Website testen. Ja, es kann nervig sein, aber besser man entdeckt seine Fehler bevor sie online ist, da die Konsequenzen später deutlich enormer sein würden. Testen sollte man Alles, woran man denken kann. Sprich man denkt sich Testfälle aus oder lässt sich aus dem Internet inspirieren, folglich müssen noch erwartete Testresultate definiert werden. Wenn diese später beim Testen erreicht werden, war jener Testfall fehlerfrei, falls nicht, muss der Fehler vermerkt und behoben werden.
